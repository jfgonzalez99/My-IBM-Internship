Removing Ant Build Dependencies

Week 1

  • Removed Ant task dependencies in database build tools files
  • Replaced with plain Java methods

Example:

import org.apache.tool.ant.Task
public class AntTask extends Task {
    public void execute() {
        log("This is an ant task.");
    }
}


public class NotAntTask {
    public void execute() {
        System.out.println("This is not an ant task.");
    }
}

Challenges

  • Unfamiliar with repository
  • Unfamiliar with Apache Ant

Lessons

  • How to navigate repository
  • A basic understanding of Ant and build process

Upgrading the Zelle Early Warning Services Emulator

Weeks 8-10

  • Utilized SOAP calls to create custom Java methods for the EWS emulator to programmatically test for fraudulent Zelle transactions
  • Allowed for the clearing of emulator data as well as the loading of external token stores, organization stores, and emulator properties to be performed via SOAP calls

Example SOAP call:

<identifier>loadtokenstore@test.com</identifier>
<person>
    <first-name>?</first-name>
    <last-name>C:\\YourDirectory\\YourTokenStore.csv</last-name>
</person>
  • Updated the SOAP traffic table to include a way to export all SOAP traffic to a text file as well as by default sort by descending date
  • Debugged an issue with retrieving a user’s pending payments and configured the emulator to behave like real EWS
  • Created help screens for Zelle Participant screen, Zelle Recipient screen, Zelle Token screen, and Zelle Organization screen

Challenges

  • Unfamiliar with SOAP
  • Passing custom file locations as a parameter to SOAP
  • Design choices (Pending Payments behavior, method of passing parameters, etc.)

Lessons

  • SOAP and SoapUI
  • Know who is using your product
  • Know what purpose your product serves
  • Design accordingly
